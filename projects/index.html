  • PipeCleaner: Manifest-Driven Self-Healing for a Cross-Machine AI Lab

    2026-07-02PythonPydanticzshlaunchdFastAPIOllamallama.cpp

    A two-machine AI lab (Mac orchestration, Windows GPU compute) failed silently: a wedged llama.cpp process exhausted the compute node's memory and took the primary 16B model offline while every status light stayed green. Repair knowledge lived in one-off scripts and operator memory.

    The failure that motivated the build is now detected by signature, repaired locally in seconds, or escalated over an allowlisted file relay with a task-ID audit trail. New builds get the full protocol for the cost of one JSON file — zero engine changes — with tiered model fallback that restores itself when the primary returns.
