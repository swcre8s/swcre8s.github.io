Self-Hosted Member Platform — Instrumented Reliability

2026-07-22FlaskSQLitecloudflaredJob queueInstrumentation

Problem
I needed to run a member-facing web platform from my own infrastructure: auth, staged onboarding, an admin surface, public exposure, and compute-heavy generation handed off to a GPU machine on the LAN. Self-hosting is easy to stand up and hard to make dependable; the failure modes are quiet, and "it works on my machine" is not a reliability claim.
What I built
A Flask/SQLite platform with session auth, staged onboarding, admin; public exposure over a cloudflared tunnel with no open ports or cloud infra; a job queue that offloads heavy generation to a GPU node over the LAN; and instrumentation throughout.
Measured results
99.5% uptime as the cloudflared tunnel uptime ratio; <250ms P95 local / ~1.2s P95 over the tunnel. The roughly one-second gap is the honest WAN cost of tunnel exposure.
What I do not claim
Solo build, measured on my own deployment, not a production system with an external user base. Code verified to compile; no end-to-end smoke pass claimed beyond that.

Why It Matters

Standing something up is the visible part. Making it dependable, measured, and honest about its limits is the systems-engineering work that separates a demo from infrastructure someone can reason about.

This case study is intentionally anonymized and decoupled from any removed or protected project. The value is the reliability pattern: keep the public surface small, instrument the pieces that fail quietly, and state the boundary instead of over-claiming.

System Shape

  • A Flask/SQLite member-facing web platform with session auth, staged onboarding, and an admin surface.
  • A cloudflared tunnel for outside access without open ports or cloud-hosted application infrastructure.
  • A queue-driven handoff from the web platform to a LAN GPU node for compute-heavy generation work.
  • Instrumentation around uptime and request latency so reliability can be discussed from measured behavior.

Measured Boundary

  • Stability99.5% uptime, measured as the cloudflared tunnel uptime ratio on the self-hosted deployment.
  • Latency<250ms P95 local / ~1.2s P95 over the tunnel. The roughly one-second delta is published because tunnel exposure has a real WAN cost.

Claim Boundary

Public boundary: this is a solo, self-hosted deployment. I am not claiming an external production user base, production scale, or an end-to-end smoke pass beyond compile verification. The project is presented as instrumented reliability work, not as a branded product.
